classless

CSS Zen Garden for standard blog-like HTML

Classless is a collection of themes for blog and blog-like websites (any webpage that follows the simple CMS content organization: pages/posts, sections/categories listing these pages/posts, a navigation menu everywhere etc.) based on pure semantic HTML.

All themes follow the same HTML structure, so you can just swap the CSS (sometimes JS) and you get a whole new theme without having to change anything. The HTML consists of only tags, no classes or ids, and makes very little assumptions about the content with which you will fill it.

If you haven't already, stop reading this and go play with the bizarre widget that is floating at the right of your screen. Choose a theme and see how this page goes with it, or choose a different HTML structure and explore the themes (this page does not implement some elements of the standard HTML, such as navs, footers and asides, also it doesn't have a "list of posts").

For theme writers

Write your theme once, constrain yourself a little to the limited space for customization of the HTML, but magically you get compatibility with all CMSes out there.

Perhaps, however, for doing extra nice stuff you'll need to use tricky CSS, which is where the fun begins, I presume.

For CMS builders

If you're writing a CMS, be it a hosted service with a specific usage or an open-source Wordpress alternative you don't need to think about themes (and all the network effect arising from it: people use if we have beautiful themes, theme builders come if we have a users) or try to be compatible with other CMSes to use themes designed for them specifically. You can just be compatible with HTML5sectioning elements and comply to this minimal standard I've come up with and you'll get automatically our pure CSS (and JS sometimes) open source themes.

You also free yourself from the burden of writing a mechanism that allow theme builders to write HTML, share it (maybe even have a "theme market") and get that HTML rendered by your software, you can accept just CSS and JS, which can be just appended to the rendered page and that's it.

Contribute!

As you can see, I'm not the best designer in the world, and all these themes are ports from random themes open-sourced on the web, so any help with theme creation and suggestions for improvements in the basic template will be very much appreciated.

There's a GitHub repository with more information, the basic template and all themes, and a playground for dinamically testing themes with somewhat-random generated HTML (remember that these random pages aren't supposed to be pretty, they're just visualization helpers). And just in case you're thinking this is a crazy new idea, well, I wasn't the first to think about it.
